# 🦙  دانلودر خودکار اولاما (اسکریپت پایدار و ضدقطعی مدل‌ها)
> 🌍 **[برای مطالعه نسخه انگلیسی کلیک کنید (English) 🇬🇧](./README.md)**

دانلود مدل‌های هوش مصنوعی حجیم در [Ollama](https://ollama.com/) با اینترنت‌های ناپایدار معمولاً به یک کابوس تبدیل می‌شود. قطعی‌های مکرر، متوقف شدن فرآیند دانلود و از همه بدتر، باگ شناخته‌شده‌ی Ollama که باعث می‌شود دانلودِ ۹۹ درصدی یک لایه حجیم پاک شده و از صفر شروع شود!

این اسکریپت Bash طراحی شده تا این فرآیند را کاملاً خودکار، ضدقطعی (Bulletproof) و هوشمند کند. 

## ✨ ویژگی‌های کلیدی

-  **تلاش مجدد خودکار (Auto-Retry):** در صورت قطع شدن اینترنت یا تایم‌اوت شدن سرور، اسکریپت بلافاصله و به صورت نامحدود فرآیند `pull` را از سر می‌گیرد تا مدل کامل شود.
-  **دور زدن باگ `partial-0` در Ollama:** گاهی پس از اتمام دانلود یک لایه حجیم، Ollama خطای `no such file` می‌دهد و باعث می‌شود کل دیتای دانلود شده از بین برود. این اسکریپت به صورت هوشمند این خطا را تشخیص داده و با ساخت یک فایل مجازی (Dummy file)، از دانلود مجدد لایه جلوگیری می‌کند.
-  **دانلود صفی و گروهی:** می‌توانید لیست بلند بالایی از مدل‌ها را به اسکریپت بدهید تا یکی پس از دیگری دانلود شوند.
-  **اجرا در پس‌زمینه (با `tmux`):** حتی اگر لپ‌تاپ خود را خاموش کنید یا ارتباط SSH شما با سرور قطع شود، دانلودها در پس‌زمینه سرور به کار خود ادامه می‌دهند.

## راهنمای راه‌اندازی و استفاده

### ۱. انتخاب مدل‌ها
ابتدا به صفحه جستجوی مدل‌های Ollama در آدرس [ollama.com/search](https://ollama.com/search) بروید و نام مدل‌های مدنظر خود را پیدا کنید (مثلاً `llama3.3:70b` یا `qwen:32b`).

سپس فایل اسکریپت (`ollma_puller.sh`) را ویرایش کرده و نام مدل‌ها را در آرایه `MODELS` وارد کنید:
```bash
MODELS=(
    "llama3.3:70b"
    "qwen3.6:35b"
    "gemma4:31b"
)
```

### ۲. نصب ابزار tmux (بسیار مهم)
برای اینکه با خاموش کردن لپ‌تاپ یا قطع شدن ارتباط اینترنت شما، فرآیند دانلود در سرور متوقف نشود، باید اسکریپت را درون محیط `tmux` اجرا کنید. اگر آن را نصب ندارید:
```bash
# Ubuntu / Debian
sudo apt install tmux

# CentOS / RHEL
sudo yum install tmux
```

### ۳. اجرای اسکریپت
ابتدا به اسکریپت دسترسی اجرایی بدهید:
```bash
chmod +x ollma_puller.sh
```

حالا یک نشست (Session) جدید در `tmux` بسازید و اسکریپت را اجرا کنید:
```bash
# 1. ساخت و ورود به محیط امن tmux
tmux new -s aipull

# 2. اجرای اسکریپت
./ollma_puller.sh
```

**نحوه خروج امن (بدون بستن دانلود):** 
حالا می‌توانید لپ‌تاپ خود را خاموش کنید! برای خروج از محیط بدون متوقف شدن اسکریپت، روی کیبورد کلیدهای `Ctrl + B` را بزنید، رها کنید و سپس دکمه `D` را بزنید.

**نحوه بازگشت و مشاهده وضعیت دانلود:**
هر زمان که دوباره به سرور متصل شدید، برای دیدن وضعیت دانلودها دستور زیر را وارد کنید:
```bash
tmux attach -t aipull
```

##  توضیح فنی: این اسکریپت چگونه باگ Ollama را حل می‌کند؟

هنگام دانلود لایه‌های بسیار سنگین، ممکن است با خطای زیر در Ollama مواجه شوید:
> `Error: remove /usr/share/ollama/.ollama/models/blobs/sha256-...-partial-0: no such file or directory`

این خطا زمانی رخ می‌دهد که فایل با موفقیت دانلود شده و تغییر نام یافته، اما تابع پاک‌کننده (Cleanup) در Ollama به دنبال نام موقت فایل می‌گردد و چون آن را پیدا نمی‌کند، کِرَش (Crash) کرده و فرآیند را از صفر شروع می‌کند.
این اسکریپت خروجی کنسول را مانیتور می‌کند. به محض مشاهده این خطا، دقیقاً در همان مسیر یک فایل خالی (`touch`) می‌سازد تا Ollama بتواند آن را "پاک" کند و باگ را دور بزند؛ این یعنی **خداحافظی با شروع مجدد دانلودهای چند ده گیگابایتی!**
